# SHIFT Finance — WCAG 2.2 AA Accessibility Audit

**URL audited**: https://shift-rwa.vercel.app/
**Pages covered**: `/`, `/team`, `/trade`, `/trade/TSL2L`, `/rewards`, `/blog`, `/blog/[slug]`
**Standard**: WCAG 2.2 Level AA
**Date**: 2026-04-24
**Auditor**: AccessibilityAuditor
**Tools**: Playwright MCP (live-site interaction), computed-style contrast calculator, manual keyboard + accessibility-tree probing, axe-equivalent heuristics run against the DOM. No deploy was touched; all evidence is live-site DOM + CSSOM data.

---

## Executive Summary

**Conformance verdict**: **DOES NOT CONFORM** with WCAG 2.2 Level AA.

The product has the bones of an accessible site (semantic header/footer, labelled nav, `lang="en"`, no SPA route-change title swap issues, good alt on blog hero images, a form textbox labelled "USDC amount"). But once you leave the happy path, barriers stack up fast:

- **Zero skip-to-main link** across every page.
- **Focus indicator on the primary Launch App CTA is literally invisible** — the outline color equals the page background color (1.00:1 contrast).
- **Seven hero & section headings are gradient-clipped text**, with a visible dark-end stop (`#07638C`) that is **2.66:1 on `#021C24`** — fails both 1.4.3 (text) and 1.4.11 (non-text) at the tail of the gradient.
- **Multiple `<main>` landmarks** on `/team`, `/blog`, and blog-post pages (up to 2 `<main>` and 2 `<header>`) — a WCAG 1.3.1 landmark violation that breaks SR landmark navigation.
- **Trade detail pages ship with no `<h1>`** (starts at h3 "What is TSL2L?") — fails 1.3.1 / 2.4.6.
- **Heavy animation layer (10+ infinite orbits, pulses, chip-orbits)** runs unconditionally; the only `prefers-reduced-motion` rule in the stylesheet is scoped to the toast library. Violates 2.3.3 and likely 2.2.2.
- **Newsletter and Rewards email inputs are completely unlabelled** — only a placeholder.
- **FAQ disclosure buttons expose `aria-expanded` but no `aria-controls`** — SR users get state but not the referenced panel.
- **Blog filter buttons (All / Signal / Academy / General)** advertise no selected state (no `aria-pressed`, no `aria-current`).
- **Footer + header text links on mobile measure ~18px tall** — fails 2.5.8 (24×24 minimum).
- **Comparison-table row headers are wrapped in `<button>`** with no handler — misuses interactive semantics and creates phantom Tab stops.

**Findings total**: 32 (6 P0 blockers, 16 P1 serious, 10 P2 minor)

---

## Methodology

| Check | How it was tested |
|---|---|
| Landmarks & headings | `document.querySelectorAll('header, main, footer, nav, h1…h6')` on every route |
| Accessible names | Accessibility-tree snapshot via `browser_snapshot`, plus text + `aria-label` + nested `<img alt>` fallback check |
| Contrast (text) | Walked every DOM node with direct text content, computed (foreground, background) pairs, applied WCAG 2.x relative-luminance formula, flagged combinations below 4.5 / 3.0 |
| Contrast (non-text) | Sampled dividers, focus outlines, button borders against resolved background |
| Focus-visible | `el.focus()` then read `outline`, `outlineColor`, `outlineOffset`, `boxShadow` for each CTA; cross-referenced against page background |
| Keyboard / tab order | DOM order + `getBoundingClientRect()` to catch out-of-viewport focusables and logical flow anomalies |
| Touch targets | Collected rects at 390×844 (iPhone 14) and 1440×900; applied 24×24 (AA) and 44×44 (AAA recommended) thresholds |
| Motion | `document.getAnimations()`, stylesheet scan for `@media (prefers-reduced-motion)` conditionText, count of `infinite` declarations |
| Forms | Every `<input>/<textarea>/<select>` checked for `<label for>`, `aria-label`, `aria-labelledby`, `aria-describedby`, `aria-invalid`, `required` |
| Duplicate link text | Grouped every `<a>` by its accessible name and flagged names pointing to more than one `href` |
| External links | `target="_blank"` pairs checked for `rel="noopener noreferrer"` |

## Findings

### P0 — BLOCKERS

#### 1. "Launch App" focus indicator is invisible against the dark page background
- **WCAG**: 2.4.7 Focus Visible (AA), 2.4.11 Focus Not Obscured (Minimum, AA new in 2.2), 1.4.11 Non-text Contrast (AA)
- **Page / element**: Every page — header right-aligned CTA `<a href="/app">Launch App`
- **Evidence**: `getComputedStyle` after `.focus()` reports `outline: rgb(2, 28, 36) none 1.5px`. The button's background is `#26C8B8` (mint). The outline color is `#021C24` — **identical to the page background**. Contrast of outline vs page background = **1.00:1**. The ring is only visible on the 1.5px strip that happens to overlap the mint pill; the portion bleeding onto the page is pure page-bg.
- **Fix**:
  ```css
  a[href="/app"]:focus-visible {
    outline: 2px solid #26C8B8;   /* mint, 8.4:1 vs page */
    outline-offset: 3px;
    box-shadow: 0 0 0 4px rgba(38, 200, 184, 0.35);
  }
  ```
  Apply the same ring token to every CTA that sits directly on `#021C24`.

#### 2. Trade detail pages have no `<h1>` and an inconsistent document title
- **WCAG**: 1.3.1 Info and Relationships (A), 2.4.2 Page Titled (A), 2.4.6 Headings and Labels (AA)
- **Page**: `/trade/TSL2L` (all `/trade/[slug]`), and `/trade` is a 404 page whose `<title>` still reads "SHIFT — 3× & 2× Bi-Directional Tokenized Stocks. No Liquidations." (home title) while the body shows a giant `404`.
- **Evidence**: `document.querySelectorAll('h1').length === 0` on `/trade/TSL2L`. First heading encountered is `<h3>What is TSL2L?`. Screen-reader heading navigation skips the primary asset name/price entirely.
- **Fix**: Render a visually hidden `<h1>TSL2L — 2× long Tesla (TSLA)</h1>` at the top of the main region, or promote the existing `TSL2L` / price stack to an `<h1>`. For `/trade` (index), set a distinct `<title>Trade — SHIFT Finance` and an actual `<h1>Trade` or a real 404 page at `/trade` with `<title>Page not found · SHIFT Finance`.

#### 3. Gradient-clipped hero headings have an illegal dark-end stop
- **WCAG**: 1.4.3 Contrast (Minimum) (AA), 1.4.11 Non-text Contrast (AA)
- **Page**: Homepage hero + "Two broken extremes", "SHIFT fixes both", "Three ways to get leverage", "Built by operators…", "Earn $SHFT tomorrow", "Trade leveraged stocks like a crypto native" — all seven h1/h2 gradient spans.
- **Evidence**: Computed `background-image: linear-gradient(135deg, rgb(38,200,184) 0%, rgb(7,99,140) 100%)` on `span.bg-clip-text.text-transparent`. Mint end `#26C8B8` = 8.40:1 on `#021C24` (passes); dark end `#07638C` = **2.66:1** on `#021C24` (fails 3:1 for large text; fails 4.5:1 for any non-large portion). Because the gradient direction is 135°, the last ~30% of every heading sits in the unreadable zone.
- **Fix**: Either (a) shift the dark stop to at least `#0E86B8` (tested ~3.1:1 minimum) and ideally to `#14A6C8` for ≥4.5:1 safe for any glyph size, or (b) drop the gradient and keep solid mint `#26C8B8`.

#### 4. Newsletter email inputs have no accessible label
- **WCAG**: 3.3.2 Labels or Instructions (A), 1.3.1 Info and Relationships (A), 4.1.2 Name, Role, Value (A)
- **Page**: `/rewards` hero capture form; `/blog` "Subscribe to the SHIFT Signal" block at the bottom; same block on every blog post.
- **Evidence**: `<input type="email" id="base-ui-_R_64lfiur3b_" placeholder="you@wallet.io">` — no `<label for>`, no `aria-label`, no `aria-labelledby`. Same pattern on `/blog` (`placeholder="your@email.com"`). VoiceOver announces only "edit text, you at wallet dot io" with no field purpose.
- **Fix**: Add a visually hidden label or an explicit `aria-label`:
  ```html
  <label for="newsletter-email" class="sr-only">Email address</label>
  <input id="newsletter-email" type="email" required
         aria-describedby="newsletter-help"
         placeholder="you@wallet.io">
  <span id="newsletter-help" class="sr-only">We'll email you when rewards unlock.</span>
  ```

#### 5. Multiple `<main>` landmarks (and, on blog posts, multiple `<header>`)
- **WCAG**: 1.3.1 Info and Relationships (A), 2.4.1 Bypass Blocks (A)
- **Page**: `/team` (2 `<main>`), `/blog` (2 `<main>`), `/blog/[slug]` (2 `<main>`, 2 `<header>`).
- **Evidence**: `document.querySelectorAll('main').length === 2` on every listed route. Likely caused by a layout wrapper and a page-level `<main>` both rendering. SR landmark jumping (NVDA `D`, VO rotor) collides.
- **Fix**: Keep exactly one `<main>` per page. Remove the outer `<main>` from the layout component, or wrap page content in `<article>` / `<section aria-labelledby>` instead.

#### 6. Zero skip-to-main-content link
- **WCAG**: 2.4.1 Bypass Blocks (A)
- **Page**: Every page.
- **Evidence**: No `a[href="#main"]`, no focusable-on-first-Tab element containing "Skip". First Tab stop is the "SHIFT home" logo link. Keyboard users pay 7 Tab presses before reaching the main content area on every single page.
- **Fix**: Add at the very top of `<body>` (before `<header>`):
  ```html
  <a href="#main-content" class="skip-link">Skip to main content</a>
  ```
  ```css
  .skip-link {
    position: absolute; top: -40px; left: 8px;
    background: #26C8B8; color: #021C24; padding: 8px 12px;
    border-radius: 6px; font-weight: 600; z-index: 9999;
  }
  .skip-link:focus-visible { top: 8px; outline: 2px solid #fff; outline-offset: 2px; }
  ```
  Make sure `<main id="main-content" tabindex="-1">` so focus actually moves.

### P1 — SERIOUS

#### 7. FAQ disclosure buttons missing `aria-controls`
- **WCAG**: 4.1.2 Name, Role, Value (A), 1.3.1 (A)
- **Page**: Homepage `Frequently asked` section.
- **Evidence**: 7 buttons have `aria-expanded="false"` but `aria-controls === null`. SR users hear "collapsed" but aren't told which region the button toggles, and can't jump to the revealed panel via the idref path.
- **Fix**: For each panel, assign an `id` and set `aria-controls` on the trigger (Base UI Accordion exposes this via `render` prop or by wiring `Accordion.Panel id=...`). Confirm the panel has `role="region"` (implicit with `<section>`) and `aria-labelledby` pointing back to the trigger id.

#### 8. Comparison-table row headers are wrapped in `<button>` with no action
- **WCAG**: 1.3.1 (A), 4.1.2 (A), 2.4.4 Link Purpose (A)
- **Page**: Homepage "Three ways to get leverage" comparison table. Rows "Short side", "Liquidation risk", "Funding fees", "Forced close".
- **Evidence**: Accessibility tree: `cell "Short side" → button "Short side"`. The button has no visible trigger icon, no `aria-describedby`, and clicking it does nothing observable. It forces a phantom Tab stop.
- **Fix**: If these are meant to open tooltips or info popovers, make that clear — add an info icon, wire the popover, and give the button an accurate name like "About short-side exposure, tooltip". If purely decorative/semantic, demote to `<th scope="row">` without the button.

#### 9. Infinite-duration decorative animations ignore `prefers-reduced-motion`
- **WCAG**: 2.3.3 Animation from Interactions (AAA) — best practice still; 2.2.2 Pause, Stop, Hide (A) — orbits run >5s with moving content near readable copy
- **Page**: Homepage hero, footer orbit ring, traction scroll, shield-pulse.
- **Evidence**: `document.getAnimations()` returns `pulse` (2s), `shift-shield-pulse` (3s), `traction-scroll` (20s), `shift-orbit-1…5` (12s–60s), `shift-chip-orbit-0…` (22s+), all `playState: running`, all infinite. Only two `@media (prefers-reduced-motion)` rules exist in the compiled stylesheet and both are scoped to the Sonner toast library. The orbit `@keyframes` are defined inside `@media (prefers-reduced-motion: no-preference)` — but the `animation: shift-orbit-1 60s infinite` property is declared outside that query, so the property still applies even when reduce is requested (the engine just uses default/placeholder keyframes).
- **Fix**: Add a global guard:
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.001ms !important;
      animation-iteration-count: 1 !important;
      transition-duration: 0.001ms !important;
      scroll-behavior: auto !important;
    }
  }
  ```
  And add a persistent UI pause control for the orbit ring (2.2.2 requires a mechanism to pause auto-updating content that runs ≥5s).

#### 10. Divider borders are below 3:1 non-text contrast
- **WCAG**: 1.4.11 Non-text Contrast (AA)
- **Page**: Every card edge, table cell border, input border — computed border color `rgb(18,54,66)` on bg `rgb(2,28,36)`.
- **Evidence**: Measured contrast **1.37:1**, requirement 3:1.
- **Fix**: Raise dividers to `#1E4F60` (~2.5:1 — still failing) or `#2A6E85` (~3.3:1 — passes). Alternatively apply dividers only where a card's background lightens the surface first.

#### 11. Duplicate "(empty)" accessible names on /team social icon links
- **WCAG**: 2.4.4 Link Purpose (A), 4.1.2 Name, Role, Value (A)
- **Page**: `/team`.
- **Evidence**: JS probe reports 6 links with empty accessible name: `https://shift-rwa.vercel.app/`, plus X, Discord, Telegram, YouTube, LinkedIn. The icons render `<img>` with empty alt and no `aria-label` on the `<a>` on this route (the home-page footer correctly exposes `aria-label`, so this is a regression in the `/team` layout).
- **Fix**: Add `aria-label="X (Twitter)"`, `aria-label="Discord"`, etc. on each social anchor. Or give the SVG child `<title>` elements that match.

#### 12. Mobile nav toggle missing `aria-expanded` / `aria-controls`
- **WCAG**: 4.1.2 (A), 1.3.1 (A)
- **Page**: Header on every page at widths <1024px.
- **Evidence**: `button[aria-label="Open navigation menu"]` — `aria-expanded: null`, `aria-controls: null`, label stays "Open navigation menu" even when the menu is already open.
- **Fix**: Wire up `aria-expanded` (toggle true/false on click), add `aria-controls="mobile-nav-panel"` pointing to the drawer `<div id="mobile-nav-panel">`, and swap the label between "Open navigation menu" ↔ "Close navigation menu" based on state.

#### 13. Nav links at x:0,y:0 remain keyboard-focusable while hidden
- **WCAG**: 2.4.3 Focus Order (A), 2.4.11 Focus Not Obscured (AA)
- **Page**: Any page at viewport <1024px — the desktop nav is `display: hidden` (probably visibility clipped) but anchor elements still have `tabindex=0`.
- **Evidence**: Tab-order probe returned 6 nav anchors at `{x:0, y:0}` between the logo and the "Launch App" CTA. Keyboard users blindly press Tab 6 times on nothing visible.
- **Fix**: Ensure the collapsed nav uses `display:none`, `hidden`, or `visibility:hidden` + `inert` on the container so its descendants drop out of the tab sequence. `tailwindcss`-style `hidden lg:flex` is correct; check that the rendered output actually applies `display:none` rather than clipping with `opacity:0`.

#### 14. Blog filter buttons do not advertise selected state
- **WCAG**: 4.1.2 (A), 1.3.1 (A), 2.4.6 (AA)
- **Page**: `/blog`.
- **Evidence**: Four category buttons (`All`, `Signal`, `Academy`, `General`) — none carry `aria-pressed`, `aria-current`, or `aria-selected`. SR users have no announced indication of which filter is active.
- **Fix**: If filters are a toggle set, add `aria-pressed="true|false"`. If they're a tablist (controlling a single filtered region), refactor to `role="tablist"` / `role="tab"` + `role="tabpanel"`. Name the active filter in the accessible name: `aria-label="Filter by Academy — selected"`.

#### 15. Text anchor links below 24×24 on mobile
- **WCAG**: 2.5.8 Target Size (Minimum) (AA — new in 2.2)
- **Page**: Every page. Header nav + footer nav + inline "See all markets", "View all", "Full team + advisors", "Back to blog" anchors.
- **Evidence**: At 390×844 viewport, anchors measure 18px tall (e.g., footer "Markets" 54×18, "Terms of Use" 87×18). 2.5.8 requires 24×24 CSS pixels for a conforming target.
- **Fix**: Raise inline link hit areas: wrap in `<li>` with `padding-block: 8px`, or set `display:inline-block; min-height: 24px; min-width: 24px; padding: 4px 0` on the anchors themselves.

#### 16. Trade ticker list items lack programmatic price-change announcement
- **WCAG**: 4.1.3 Status Messages (AA), 1.3.1 (A)
- **Page**: `/trade/TSL2L` (rate readout "Rate / 21.53 USDC = 1 TSL2L", "Est. receive"), and the home-page ticker cards ("+2.41% TSL2L $175.60").
- **Evidence**: Numbers change as the live rate refreshes, but the wrapper is a plain `<div>` with no `aria-live`. SR users hear nothing.
- **Fix**: Wrap the dynamic readout in `<div aria-live="polite" aria-atomic="true">` so polite updates fire; keep the TradingView chart iframe aside since it has its own a11y.

#### 17. Hero stat cards — "$2M Seed Closed", "$40M+ Liquidity Raised" — have no heading/label association
- **WCAG**: 1.3.1 (A), 2.4.6 (AA)
- **Page**: Homepage hero and CTA band.
- **Evidence**: Each stat is two plain `<div>`s, one number one caption. SR users hear "dollar 2 M" then "seed closed" with nothing relating the two. No `<dl>`, no heading, no `aria-describedby`.
- **Fix**: Use a description list:
  ```html
  <dl>
    <dt>Seed</dt><dd>$2M raised</dd>
    <dt>Liquidity</dt><dd>$40M+</dd>
  </dl>
  ```

#### 18. Chainlink Proof-of-Reserves link opens in new tab without an indicator for SR users
- **WCAG**: 3.2.5 Change on Request (AAA) / G201 technique (AA practice)
- **Page**: Homepage hero; also dozens of investor / integration links.
- **Evidence**: Every `a[target="_blank"]` has `rel="noopener noreferrer"` (good), but none exposes "opens in a new tab" in the accessible name, nor a visible icon with `<title>`.
- **Fix**: Append a visually-hidden span: `<span class="sr-only"> (opens in a new tab)</span>`, or a decorative `↗` icon with an explicit `aria-label="opens in a new tab"`.

#### 19. TSL2L trade pair buttons (Buy/Sell) not implemented as a tablist / radiogroup
- **WCAG**: 1.3.1 (A), 4.1.2 (A), 2.4.6 (AA)
- **Page**: `/trade/TSL2L`.
- **Evidence**: The Buy/Sell toggle is two sibling `<button>` elements. No `role="tablist"`, no `aria-selected`, no arrow-key keyboard pattern between them. SR users can't tell which side is active.
- **Fix**: Model as a WAI-ARIA Tabs pattern or a radio group. Minimal fix: `role="group" aria-label="Trade direction"` on the wrapper and `aria-pressed="true|false"` on each button.

#### 20. Redundant alt text on blog hero image duplicates the `<h1>` verbatim
- **WCAG**: 1.1.1 Non-text Content (A) (best-practice)
- **Page**: Every `/blog/[slug]`.
- **Evidence**: Hero `<img alt="The SHIFT Signal #1: Wall Street Doesn't Want You to Know This">` is directly adjacent to `<h1>The SHIFT Signal #1: Wall Street Doesn't Want You to Know This</h1>`. VoiceOver announces the title twice.
- **Fix**: Mark the hero as decorative — `alt=""` — since the title is already the H1. Reserve descriptive alt for editorial illustrations that carry unique meaning.

#### 21. Blog dates rendered as plain text, not `<time>` with `datetime`
- **WCAG**: 1.3.1 (A)
- **Page**: `/blog`, every blog post.
- **Evidence**: Probe reports `document.querySelectorAll('time').length === 0` on both pages. Dates like "Jan 14, 2026" and "7 min" are emitted as text inside `<div>`s.
- **Fix**: `<time datetime="2026-01-14">Jan 14, 2026</time>` — helps SR, indexing, and relative-time parsers. Do the same for read-time durations or mark them with `aria-label="7-minute read"` on the surrounding element.

#### 22. Document titles carry a double brand suffix on many routes
- **WCAG**: 2.4.2 Page Titled (A) (best practice)
- **Pages**: Every non-home page — e.g. `Team | SHIFT · SHIFT Finance`, `The SHIFT Signal #1… — SHIFT Finance · SHIFT Finance`, `TSL2L · TSLA long 2 · SHIFT Finance`.
- **Evidence**: "SHIFT" appears twice in the title tag on /team, blog detail; inconsistent separator (`·` vs `|` vs `—`).
- **Fix**: Settle on one template — e.g. `{Page title} · SHIFT Finance` — and kill the duplicate "SHIFT" / "SHIFT Finance" trailing chain.

### P2 — MINOR

#### 23. `<html>` missing `dir="ltr"`
- **WCAG**: 3.1.1 Language of Page (A) (dir is recommended even if implicit)
- Every page: `document.documentElement.dir === ""`.
- Fix: `<html lang="en" dir="ltr">` for explicit directionality — matters when you start adding Arabic / Hebrew blog localization.

#### 24. No `viewport` `maximum-scale=1` is fine, but `user-scalable` should be explicit
- Current: `width=device-width, initial-scale=1, maximum-scale=5` — acceptable, but the cap at 5 is below the 500% zoom commonly used.
- WCAG 1.4.4 Resize Text (AA) — passes, but AAA 1.4.10 Reflow recommends unlimited.
- Fix: drop `maximum-scale=5` or raise to 10.

#### 25. Footer "All systems operational" carries no relationship to the /status link
- Page: Every page footer.
- Two sibling `<div>`s: one says "© 2026 SHIFT Finance…" and a separate one says "All systems operational" in green. No `<a href="/status">`, no live-region, no status role.
- Fix: `<a href="/status" class="status-chip" aria-label="System status: all systems operational">All systems operational</a>`.

#### 26. `aria-label="SHIFT home"` on the logo link is duplicated by child `<img alt="SHIFT Finance">`
- Page: Header on every page.
- Result: SR announces "SHIFT home SHIFT Finance link" — mild clutter, not a blocker.
- Fix: Keep `aria-label` on the link, set `alt=""` on the logo image.

#### 27. Blog "View all" / "Full team + advisors" / "See all markets" — link purpose is ambiguous when tabbing out of context
- WCAG 2.4.4 (A) passes literally (context), 2.4.9 AAA would fail.
- Fix: make each anchor self-describing: "View all blog posts", "See full team and advisors", "See all markets".

#### 28. Price-change rows in the markets grid rely on color alone
- Page: Homepage "Top Movers", ticker list.
- Green "+2.41%" and red "-2.41%" are colored icons (up/down arrows) plus colored text. The arrow svgs carry no `alt`/`<title>`, so for colorblind SR users reading the non-visual output, direction may be ambiguous when the minus sign is missed.
- WCAG 1.4.1 Use of Color (A).
- Fix: Add `aria-label="+2.41%, up"` to the combined group; or a visually hidden "down by" prefix next to negative values.

#### 29. Table captions missing on the "Three ways to get leverage" comparison
- WCAG 1.3.1 (A) — semantic table exists, but no `<caption>`.
- Fix: `<caption class="sr-only">Comparison of spot stocks, perps, and SHIFT leveraged tokens</caption>` right after `<table>`.

#### 30. Forms do not expose `aria-invalid` / error live-region scaffolding for future state
- WCAG 3.3.1 Error Identification (A), 3.3.3 Error Suggestion (AA).
- Page: Newsletter form on every page.
- Today it submits optimistically without validation errors visible. When you add validation, the current input has no `aria-describedby` wired to an error id.
- Fix: Pre-wire `<p id="newsletter-error" role="alert" hidden>…`, set `aria-describedby="newsletter-error"` on the input, and flip `aria-invalid="true"` + `hidden` off on error.

#### 31. Market cards link accessible name contains redundant ticker twice
- Page: Homepage, `/markets`.
- Example: `link "TSL2L 2× long TSL2L TSLA long 2 $175.60 +2.41%"` — "TSL2L" appears twice. The image alt plus the visible ticker label both feed into the accessible name.
- WCAG 2.4.4 (A) passes; 2.4.6 (AA) borderline.
- Fix: `<img alt="">` on the ticker emblem since the ticker text is adjacent, or use `aria-labelledby` to craft a clean name: "TSL2L, 2× long Tesla, $175.60, up 2.41%".

#### 32. Alert container at `ref=e219/e1084/e121` has empty accessible name
- Page: Every page — a `<div role="alert">` is rendered empty (likely for future Sonner toasts) at the bottom of the DOM.
- WCAG 4.1.2 (A): role="alert" with no accessible name is redundant noise but not a failure until a toast fires.
- Fix: Keep it empty until content is appended; confirm Sonner properly sets focus and polite/assertive live-region semantics.

## Positive findings (preserve these)

- `<html lang="en">` is set globally.
- Header includes `navigation "Main navigation"` — landmarks labelled.
- The USDC amount input on `/trade/TSL2L` has a proper accessible name.
- All `target="_blank"` links carry `rel="noopener noreferrer"`.
- Blog post content uses a single clean `<h1>` and a well-structured `<h2>` outline.
- Most images on the home page have meaningful `alt` (ticker names, team members, partner logos) — no missing-alt failures detected on / or /team hero.

## Recommended next steps

1. Add `@axe-core/react` or `vitest-axe` to the CI suite so any new component regression is caught during PR review.
